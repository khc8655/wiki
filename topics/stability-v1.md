## Exclude unless user explicitly asks
- 安全机制（加密、认证、权限等）- 应路由到 security 主题
- 纯运维监控细节（除非用户询问大数据运维）
- 与稳定性无关的网络架构细节

## Retrieval advice

1. 查询包含"多活"、"热备"、"容灾"、"可靠性"时，优先命中 sec-234~237
2. 查询包含"抗丢包"、"弱网"、"网络适应性"时，优先命中 sec-010 和抗丢包算法相关卡片
3. 查询包含"扩容"、"弹性"时，优先命中 sec-241~243
4. 查询包含"稳定性"、"高可用"、"可靠"时，优先命中平台稳定性总览卡片
5. 回答时展示每条候选的匹配度，并区分"强命中 / 弱相关 / 排除项"
