**1.1 平台默认密码统一**

目前，风铃 Console 平台、VMS 本地 web 页面和生态服务器本地 web 页面默认密码均不统一 ，不便于用户使用。因此，本期实现密码统一化，并增加强制修改密码逻辑。

密码统一

l

用户部署新版本或新工厂包，平台默认密码均统一为 Xy@system0；

用户从现有旧版本升级到新版本，或从新版本回退到旧版本，不影响用户现有密码继续使用；

密码强制修改

l

首次缺省 IP 访问生态服务器本地 web 页面时，输入默认密码后，强制修改密码；修改密码成功后，才能重新登陆 web，进入首页。

**2.2.1 轮询支持终端自动填充**

在实际使用场景下，多个项目反馈使用自动轮询时，最后一屏的显示画面由于人数不够，会导致出现空窗口，影响会议体验。因此本期在执行终端侧的轮询操作时支持终端画面自动填充机制 ，规避空窗口显示问题。

支持的终端型号： PE8000-A ，PE7000

在专业会控界面，选择待执行轮询操作的终端，打开轮询界面。本期根据终端能力新增两个设置项：**循环轮询模式**和**终端自动填充**。

l 循环轮询模式

在以下配置下，根据终端上报能力，有能力的终端显示**循环轮询模式**配置项：

• 自动轮询模式下，当**轮询列表**设置为**指定终端**时，显示**循环轮询模式**配置开关。

• 混合轮询模式下，当**自动轮询成员列表**设置为**指定终端**时，显示**循环轮询模式**配置开关。

开启**循环轮询模式**开关后，自动轮询按指定终端列表首尾相接执行轮询操作，不区分

第一页和最后一页。

![](data:image/png;base64...)

l 终端自动填充

根据终端上报能力，有能力的终端显示**终端自动填充**配置项：

• 自动轮询模式下，当**轮询列表**设置为**所有终端**时，**终端自动填充**模式可配置为**空窗口**。勾选后，最后一屏显示出现的空窗口将随机被会中成员填充；

当最后一屏显示无法填满所有窗口时，将自动填充未被过滤且与当前画面不重复的成员（即前面轮询过的成员）到当前轮询画面的空窗口中，若会中无更多不重复成员可填充，则不用再自动填充保持空窗口显示。

![](data:image/jpeg;base64...)

• 自动轮询模式下，当**轮询列表**设置为**指定终端**时，**终端自动填充**模式可配置为**空窗**

**口**。勾选后，最后一屏显示出现的空窗口将随机填充指定终端列表中的成员。

自动填充的成员必须是指定轮询列表的成员，优先去掉第一页和倒数第二页的成员（避免和上一屏及下一屏成员重复）， 然后用其他成员进行填充，如果没有更多页数，则无要求随机填充。

![](data:image/png;base64...)

• 手动轮询模式下，**终端自动填充**模式可配置为**轮询过滤跳过的成员**。勾选后，当成员满足过滤条件被跳过时，该成员位置自动被窗口轮询列表的其他人员进行填充。

![](data:image/jpeg;base64...)

• 混合轮询模式下，**终端自动填充**模式可配置为**空窗口**或**轮询过滤跳过的成员**。

![](data:image/png;base64...)
