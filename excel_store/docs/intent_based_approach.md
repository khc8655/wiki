# 意图驱动的知识库查询（大胆方案）

## 核心思想

**不按数据来源分类，按用户意图分类。**

用户不关心数据存在哪个Excel或文档里，只关心：
- 我想知道什么？
- 我想做什么？

## 新分类：4类意图

### 1. Factual（事实查询）
**用户想问："是什么/有多少/什么时候"**

- AE800多少钱？（价格事实）
- PE8000什么时候停产？（时间事实）
- AE800有几个HDMI口？（规格事实）

**数据源**：pricing + comparison + proposal 中提取的事实

**返回格式**：直接给出事实，附证据

---

### 2. Comparative（对比查询）
**用户想问："有什么区别/哪个更好"**

- XE800和AE800有什么区别？
- 10方和25方会议室哪个划算？
- 分体式和一体式终端怎么选？

**数据源**：comparison 矩阵 + pricing 差异 + release_notes 功能差异

**返回格式**：对比表格 + 差异总结

---

### 3. Narrative（叙事查询）
**用户想问："介绍一下/是什么/有什么优势"**

- AE800是什么产品？
- 小鱼易连架构有什么优势？
- 2026年3月迭代了什么？

**数据源**：solution + release_notes + proposal 描述

**返回格式**：段落描述 + 要点列表

---

### 4. Procedural（过程查询）
**用户想问："怎么做/如何配置/步骤是什么"**

- 怎么配置双机热备？
- 招标参数怎么写？
- 如何申请停产替代？

**数据源**：solution 配置章节 + proposal 阶段描述 + release_notes 操作说明

**返回格式**：步骤列表 + 注意事项

---

## 实现方式：意图识别 + 统一检索

```
用户查询
    ↓
意图分类器（规则/轻量模型）
    ↓
    ├─ Factual → 检索所有数据源中的事实字段
    ├─ Comparative → 检索对比相关数据
    ├─ Narrative → 检索描述性文本
    └─ Procedural → 检索操作步骤
    ↓
结果组装（按意图模板）
    ↓
返回给用户
```

## 优势

| 维度 | 5类数据源方案 | 4类意图方案 |
|------|-------------|------------|
| 用户理解 | 需要知道数据在哪 | 直接问问题 |
| 扩展性 | 新增数据源要改路由 | 新增数据自动归类 |
| 覆盖率 | 模糊查询可能漏 | 按意图补全 |
| 结果质量 | 单源结果 | 多源融合 |

## 更激进的方案：不分意图，纯语义

**思路**：所有数据统一向量化，用户提问直接语义检索，Top-K结果直接给模型。

```
用户：AE800和XE800哪个好？
系统：
1. 语义检索："AE800 XE800 对比 差异"
2. 拿到混合结果（comparison矩阵 + pricing差异 + release_notes功能差异）
3. 直接给模型："基于以下信息回答..."
4. 模型生成对比回答
```

**优点**：
- 无需分类
- 无需路由
- 自然语言问答

**缺点**：
- 依赖模型能力
- 检索质量要求高
- 成本较高

## 推荐方案：混合模式

**第一层**：意图识别（Factual/Comparative/Narrative/Procedural）
**第二层**：意图驱动的多源检索
**第三层**：结果融合 + 模板化输出

这样既有结构化控制，又有灵活性。

## 具体改造计划

1. **数据预处理**：把5类数据统一抽取为"实体-属性-值"三元组
2. **意图分类器**：规则 + 轻量模型
3. **检索层**：按意图选择检索策略
4. **生成层**：意图模板 + 模型润色

## 示例改造

**原查询流程**：
```
"AE800参数" → 识别为proposal → 查proposal表 → 返回招标描述
```

**新查询流程**：
```
"AE800参数" → 识别为Factual意图 → 检索所有AE800的事实属性
   ├─ pricing: 价格、单位
   ├─ comparison: 规格参数
   └─ proposal: 招标参数
→ 合并为结构化回答
```

**用户获得**：
```
【AE800 基本信息】
价格：138,000元/套
规格：10路输入、3路输出、4K能力
招标要点：分体式、国产芯片、12倍变焦...
```

---

这个方案的核心是：**从"数据源驱动"转向"意图驱动"**。
