# 知识库五类数据查询区分策略

## 五类数据定义

| 类别 | 数据源 | 内容特点 | 典型关键词 |
|------|--------|----------|-----------|
| **solution** | 原始白皮书/方案文档 | 架构设计、技术方案、优势说明 | 架构、优势、方案、原理、设计 |
| **release_notes** | 迭代功能文档 | 新功能、变更、优化、修复 | 迭代、新功能、升级、优化、修复、支持、新增 |
| **pricing** | 报价表Excel | 价格、型号、编码、停产信息 | 价格、报价、多少钱、费用、停产、替代、会议室、AI |
| **comparison** | 产品对比Excel | 功能对比、能力矩阵、参数对比 | 对比、比较、区别、差异、vs |
| **proposal** | 阶段描述Excel | 招标参数、阶段话术、描述模板 | 招标、投标、参数、询价、阶段描述 |

## 路由优先级（由高到低）

```
1. comparison (对比) - 最高优先级
   触发条件: 包含"对比/比较/区别/vs" 或 两个产品型号
   
2. proposal (招标参数) - 次高优先级
   触发条件: 包含"招标/投标/参数/询价" + 产品型号
   
3. pricing (价格) - 第三优先级
   触发条件: 包含价格词 或 产品型号
   
4. release_notes (更新文档) - 第四优先级
   触发条件: 包含迭代词 或 产品型号+功能词
   
5. solution (方案) - 兜底
   触发条件: 语义搜索
```

## 查询意图与数据类型匹配表

| 你的问题类型 | 应该命中 | 示例查询 | 返回内容 |
|-------------|---------|---------|---------|
| 查价格/费用/多少钱 | pricing | "AE800多少钱" | 价格、单位、备注、停产信息 |
| 查产品功能差异/对比 | comparison | "XE800和AE800对比" | 功能矩阵对比表 |
| 查招标参数/投标描述 | proposal | "AE800招标参数" | 三阶段描述（询价/方案/招投标） |
| 查产品新功能/迭代 | release_notes | "AE800支持哪些新功能" | 迭代版本功能说明 |
| 查技术架构/方案 | solution | "小鱼架构优势" | 技术白皮书段落 |

## 快速区分口诀

```
问"多少钱" → pricing
问"有什么区别" → comparison
问"招标写什么" → proposal
问"有什么新功能" → release_notes
问"技术方案" → solution
```

## 边界情况处理

### 歧义查询自动策略

| 查询 | 路由选择 | 原因 |
|------|---------|------|
| "AE800参数" | proposal | "参数"优先于型号匹配 |
| "AE800价格" | pricing | "价格"明确 |
| "AE800" | pricing | 型号默认走pricing |
| "AE800和XE800" | comparison | 两个型号默认对比 |
| "AE800有哪些功能" | release_notes | "功能"词触发 |
| "AE800架构" | solution | "架构"词触发 |

### 多关键词查询

如果同时包含多个类别关键词，按优先级路由：
- "AE800价格对比" → comparison（对比优先）
- "AE800招标价格" → proposal（招标优先）

## 当前路由代码逻辑

```javascript
function detectQueryType(query) {
  // 1. 对比查询最优先
  if (包含"对比/比较/vs" 或 两个型号) {
    return 'comparison';
  }
  // 2. 招标参数次之
  else if (包含"招标/投标/参数" 且 有型号) {
    return 'proposal';
  }
  // 3. 价格查询
  else if (包含"价格/多少钱/费用" 或 有型号) {
    return 'pricing';
  }
  // 4. 更新文档
  else if (包含"迭代/新功能/升级") {
    return 'release_notes';
  }
  // 5. 兜底语义搜索
  else {
    return 'solution';
  }
}
```

## 查询建议

为了提高准确率，建议提问时：

1. **明确价格意图**: "AE800的价格" / "AE800多少钱"
2. **明确对比意图**: "AE800和XE800对比" / "AE800与XE800区别"
3. **明确招标意图**: "AE800招标参数" / "AE800投标描述"
4. **明确功能意图**: "AE800支持哪些新功能" / "AE800迭代功能"

如果自动路由不准确，可以直接指定数据源：
```bash
# 直接查询特定数据源
python3 scripts/query_excel_knowledge.py "AE800" -t pricing
python3 scripts/query_excel_knowledge.py "AE800" -t comparison
python3 scripts/query_excel_knowledge.py "AE800" -t proposal
```
