# 简单召回策略：宁多勿少

## 核心原则

> **不要复杂路由，全量召回，让模型筛选。**

## 实现方式

### 1. 全源并行搜索

用户提任何问题，同时搜索5类数据源：

```
用户：AE800多少钱？
      ↓
[并行搜索]
  ├─ pricing: 价格、套装信息
  ├─ comparison: 规格参数
  ├─ proposal: 招标描述
  ├─ release_notes: 功能特性
  └─ solution: 架构方案
      ↓
[合并结果] → 全部返回
```

### 2. 简单排序规则

召回结果按相关度排序，不做复杂过滤：

```
排序依据：
1. 型号完全匹配 → 排最前
2. 型号部分匹配 → 其次
3. 关键词匹配 → 再次
4. 其他 → 最后
```

### 3. 截断策略

每类数据源最多返回Top-5，避免过多：

```
总结果数 ≤ 25条（5类×5条）
```

## 对比两种方案

### 复杂路由方案（之前）

```
分析意图 → 选择数据源 → 单源查询 → 返回结果
```

**问题**：
- 意图识别错了，结果就没了
- 用户问"AE800怎么样"，可能只查comparison，漏了pricing

### 简单召回方案（现在）

```
全源查询 → 合并排序 → 返回Top-N → 模型处理
```

**好处**：
- 不会漏
- 实现简单
- 模型有上下文，能自己挑

## 示例

**用户问**："AE800怎么样？"

**召回结果**（简化）：
```
[pricing] AE800套装价格138000元
[comparison] AE800场景定位：大型会议室终端
[comparison] AE800视频输入：10路
[proposal] AE800招标参数：分体式、国产芯片...
[release_notes] AE800支持4K、双流
[solution] 分体式终端架构优势...
```

**模型收到后**：
- 有价格、有规格、有招标参数
- 自己组织语言回答"AE800是大型会议室终端，价格13.8万，支持10路输入..."

## 代码实现

```python
def query_all_sources(query):
    """全源查询，宁多勿少"""
    results = []
    
    # 并行查5类数据源
    for source in ['pricing', 'comparison', 'proposal', 'release_notes', 'solution']:
        hits = search(source, query, top_k=5)
        for hit in hits:
            hit['_source'] = source
            results.append(hit)
    
    # 简单排序（型号匹配优先）
    results.sort(key=lambda x: x['_score'], reverse=True)
    
    # 截断返回
    return results[:15]  # 最多15条
```

## 关键认知

**不要替用户/模型做太多决定**。

- 复杂路由 = 提前帮用户筛选，可能筛错
- 全量召回 = 把所有相关信息给模型，让它自己选

模型比路由规则更聪明，让它决定用什么信息回答。

## 一句话总结

> **宁可召回100条让模型挑，也不要漏掉1条关键信息。**
